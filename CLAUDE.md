# CLAUDE.md — lyrical-inventory

## After every change
End with 2–4 "Next moves" suggestions, ranked best-first. Each is one line: what, why, effort. Offer to do the top one. Skip the list if nothing is genuinely worth doing.

## Stack constraints
- Vanilla JS, no build step
- Firebase Firestore, static hosting on GitHub Pages
- Mobile-first PWA with offline POS and later sync

## Pull Requests
- When asked for a new PR: push and create it immediately, no investigation
- After a PR is merged, start the next change on a new branch with a new PR

## General
- Prefer action over investigation when intent is clear
- Only ask clarifying questions if genuinely ambiguous
